## 📐 异常状态统一标准 / Unified Status Standards

> **重要**: 所有报告类型必须遵循 `references/status_standards.md` 中定义的统一标准

### 状态图标速查 / Status Icons Quick Reference

| 异常级别 | 图标 | 显示格式 | 适用范围 |
|----------|------|----------|----------|
| **0级 正常** | (无标记) | `数值` | 所有报告 |
| **I级 轻度** | ⚠️ | `⚠️数值↓` / `⚠️数值↑` | 血常规/血脂 |
| **II级 中度** | 🟠 | `🟠数值↓↓` / `🟠数值↑↑` | 血常规/血脂 |
| **III级 重度** | 🔴 | `🔴数值↓↓↓` / `🔴数值↑↑↑` | 所有报告 |
| **IV级 危急** | 🚨 | `🚨数值↓↓↓↓` | 血常规危急值 |

> **核心规则**: 正常指标不加任何图标或✅，仅显示数值！异常时才添加图标+方向箭头！

### 趋势箭头规范 / Trend Arrows

| 箭头 | 含义 | 使用场景 |
|------|------|----------|
| ↑ | 高于正常上限 | 数值异常偏高时 |
| ↓ | 低于正常下限 | 数值异常偏低时 |
| ↗ | 较上次升高 | 与历史对比 |
| ↘ | 较上次降低 | 与历史对比 |

### CT影像专用状态 / Imaging-Specific Status

| 评价 | 图标 | 关键词示例 |
|------|------|-----------|
| 稳定/好转 | ✅ | 稳定、好转、缩小、吸收 |
| 稳定但关注 | ⚠️ | 稳定但有炎症增多、新发病变 |
| 进展/恶化 | 🔴 | 进展、增大、转移 |

**完整标准文档**: [references/status_standards.md](./references/status_standards.md)

## ⚠️ 危急值警示 / Emergency Warning Signs（最高优先级）

**出现以下情况需立即就医 / Immediate medical attention required if:**

| 危急值 | 风险描述 |
|--------|----------|
| ANC < 0.5×10⁹/L | 重度中性粒细胞减少，高感染风险 |
| PLT < 20×10⁹/L | 出血危象，自发性出血风险 |
| Hb < 60 g/L | 重度贫血，心脏并发症风险 |
| 体温 ≥ 38.0℃ + 低ANC | 发热性中性粒细胞减少症，医疗急症 |
| 活动性出血 | 牙龈出血、鼻出血、便血/血尿 |

> **处理原则**: 发现上述情况立即联系主治医生或前往急诊！

## 支持的报告类型 / Supported Report Types

### 1. 血常规 (Blood Routine)

**核心指标 / Core Indicators：**

| 指标 | 中文名称 | 正常范围(成人) | 单位 |
|------|----------|---------------|------|
| WBC | 白细胞 | ≥4.0 | ×10⁹/L |
| ANC | 中性粒细胞 | ≥1.5 | ×10⁹/L |
| PLT | 血小板 | ≥100 | ×10⁹/L |
| Hb | 血红蛋白 | 男≥120, 女≥110 | g/L |
| CRP | C反应蛋白 | ≤10 | mg/L |
| LYM# | 淋巴细胞计数 | ≥1.1 | ×10⁰/L |

### 2. 血脂 (Lipid Profile)

**关注指标：**
- 总胆固醇 (TC)
- 甘油三酯 (TG)  
- 高密度脂蛋白 (HDL-C)
- 低密度脂蛋白 (LDL-C)

### 3. 肿瘤标志物 (Tumor Markers)

**关注指标 / Key Indicators：**

| 指标 | 中文名称 | 正常范围 | 单位 |
|------|----------|---------|------|
| CA-125 | 糖类抗原125 | 0－35 | U/mL |
| CYFRA21-1 | 细胞角蛋白19片段 | 0-3.3 | ng/mL |
| VEGF | 血管内皮生长因子 | 0-160 | pg/mL |
| SCC | 鳞状细胞癌相关抗原 | 0-3 | ng/ml |

**状态标识：**
- ✅ 正常范围内 → 治疗有效，继续监测
- 🟠 轻度升高 → 关注趋势，定期复查
- 🔴 明显升高 → 需关注病情进展

### 4. CT/MRI影像报告 (Imaging Reports)

**记录内容：**
- 检查日期、医院
- 原发灶情况
- 转移灶变化
- 总体评价
- 医生建议

## 核心工作流 / Core Workflow

### 第一步：接收并识别报告图像 / Receive & Recognize Report

当用户上传医疗报告图片时：

1. **识别报告类型**
   - 通过标题和指标名称判断是哪种报告
   - 血常规、血脂、肿瘤标志物、CT/MRI等

2. **提取关键信息**
   - 检查日期
   - 医院/实验室名称
   - 所有检测指标及数值
   - 参考范围（如有显示）

3. **数据提取最佳实践**
   - 使用视觉能力准确读取数字
   - 注意单位（WBC/ANC/PLT通常为×10⁹/L，Hb为g/L）
   - 图像不清晰时明确说明，不猜测或编造数据

### 第二步：基于参考文档评估异常 / Evaluate Using Reference Document

使用 `references/blood_test_reference.json` 作为**唯一权威依据**判断异常。

**分级标准 / Grading System：**

| 级别 | 含义 | 处理优先级 |
|------|------|-----------|
| 0级 | 正常 | 无需处理 |
| I级 | 轻度异常 | 观察监测 |
| II级 | 中度异常 | 建议就诊 |
| III级 | 重度异常 | 紧急就医 |
| IV级 | 极重度 | 立即就医 |

**特殊注意事项：**
- **Hb（血红蛋白）**：需根据性别判断 - 男性≥120g/L，女性≥110g/L
- **ANC（中性粒细胞）**：<0.5×10⁹/L为感染高风险阈值
- **PLT（血小板）**：<20×10⁹/L为出血危象阈值

### 第三步：生成综合报告 & 自动录入 / Generate Report & Auto-Record

#### 📊 指标分析表模板 / Indicator Analysis Table

```
| 指标名称 | 检测值 | 参考范围 | 异常级别 | 状态 |
|---------|-------|---------|---------|------|
| [Name]  | [Value]| [Range] | [Level]  | [Status]|
```

**状态标识 / Status Indicators（遵循统一标准）：**
- **0级 正常 Normal** → 仅显示 `数值`（**禁止加✅或任何图标！**）
- ⚠️ I级轻度异常 Mild → `⚠️数值↓` / `⚠️数值↑`
- 🟠 II级中度异常 Moderate → `🟠数值↓↓` / `🟠数值↑↑`
- 🔴 III级重度异常 Severe → `🔴数值↓↓↓` / `🔴数值↑↑↑`
- 🚨 IV级极重度 Critical → `🚨数值↓↓↓↓` (需立即关注)

#### 💡 处理建议 / Handling Recommendations

对每个异常指标提供：
1. **处理原则 Treatment Principles**
2. **出院医嘱 Discharge Orders** - 用药指导
3. **自我监测 Self-Monitoring** - 症状观察要点
4. **生活护理 Life Care** - 饮食活动注意

| 指标 | 检测值 | 参考范围 | 状态 |
|------|-------|---------|------|
| [指标1] | [值]  | [范围]  | ✅正常 |
| ...   |       |         |      |

💡 温馨提示：
1. 继续保持良好的生活习惯
2. 注意营养均衡，保证充足休息

4. 如出现不适症状及时就医

### 第四步：更新飞书文档台账 / Update Feishu Spreadsheet

> **⚠️ 核心约束（必须严格遵守，否则导致数据混乱）**

#### 4.0 写入前必做：读取现有数据（最重要！）

**在写入任何数据之前，必须先执行以下步骤：**

1. **读取目标工作表的现有数据** - 调用 `feishu_api.py` 或直接查看表格，获取：
   - 表头行（第1行）- 确认列顺序
   - 所有已有数据行 - 确认最新日期和数据格式
   - 当前总行数 - 确定插入位置

2. **确认插入位置规则（时间倒序）**
   - 新数据必须按 **日期从新到旧** 排列
   - 最新记录在第2行（表头下方），最老的在最后
   - 插入前先对比日期，确定正确位置

3. **严格匹配既有格式**
   - 逐列对照表头和已有数据的格式
   - 图标使用、文字风格、内容长度保持一致
   - 绝不创造新的格式风格

#### 4.1 时间顺序强制规范

| 规则 | 说明 | 违规后果 |
|------|------|----------|
| **日期倒序排列** | 最新数据在表头下方(第2行)，越往下日期越旧 | 数据混乱，趋势分析错误 |
| **写入前查重** | 检查相同日期+医院的记录是否已存在 | 产生重复数据 |
| **不可覆盖旧数据** | 只追加或按序插入，绝不删除/覆盖历史记录 | 历史数据丢失 |

**正确的数据排列示例（CT报告）：**
```
行号 |    A日期    |   B医院     | C总体评价  | ...
-----|-------------|-------------|------------|-----
  1  |    日期     |    医院     | 总体评价   | ... (表头)
  2  | 2025-11-03  | 医科院肿瘤  | ✅整体病情稳定 | ... (最新)
  3  | 2025-06-27  | 医科院肿瘤  | 🔴治疗后复查 | ...
  4  | 2025-06-19  | 医科院肿瘤  | ✅免疫治疗后 | ...
  5  | 2025-03-07  | 中国医学科学院| 颈+胸+腹CT平扫| ... (最老)
```

#### 4.2 列格式统一化强制要求

**CT报告各列的标准化格式：**

| 列名 | 格式要求 | 正确示例 | 错误示例 |
|------|----------|----------|----------|
| **日期** | YYYY-MM-DD | 2025-06-27 | 2025年6月27日 / 6.27 |
| **医院** | 简称，与历史一致 | 医科院肿瘤医院 | 北京医科院肿瘤医院(不同写法) |
| **检查类型** | 简短描述 | 颈+胸+腹CT平扫 | (过长描述) |
| **原发灶** | [图标]简短描述 | ✅原发灶稳定无变化 | (纯文字无图标) |
| **转移灶** | [图标]简短描述或"无" | 左肺结节大小约0.6cm；均稳定 | (散乱长文本) |
| **总体评价** | **仅图标+8字内短语** | ✅整体病情稳定 | ✅免疫治疗后部分炎性病变需有(太长) |
| **医生建议** | 简短可操作建议 | 继续定期监测 | (整段复制报告原文) |

> **关键规则**: "总体评价"列必须是 **图标 + 8个字以内的评价短语**，禁止塞入完整诊断描述！

#### 4.3 数据录入标准流程

```
┌─────────────────────────────────────────────────────┐
│                  数据录入检查清单                     │
├─────────────┬───────────────────────────────────────┤
│ □ 读取表头  │ 确认列顺序，知道每列该填什么           │
│ □ 读取历史  │ 确认最新日期、格式风格                 │
│ □ 确定位置  │ 根据日期确定插入到第几行               │
│ □ 格式对齐  │ 每列内容与历史数据格式一致             │
│ □ 长度控制  │ 总体评价≤8字，各列不过长              │
│ □ 图标一致  │ 使用status_standards.md规定的图标      │
│ □ 执行写入  │ 调用API插入到正确位置                 │
│ □ 结果验证  │ 确认写入成功且格式正确                 │
└─────────────┴───────────────────────────────────────┘
```

写入CT报告时，row_data 必须严格按照 config.json 中 ct_report.columns 的顺序：

## 最佳实践 / Best Practices

### 准确性与安全性 / Accuracy and Safety

1. **绝不编造数据 Never Fabricate Data**
   - 图片不清晰时明确说明无法读取的内容
   - 关键信息缺失时请用户补充

2. **仅使用参考文档 Use Only the Reference Document**
   - 所有异常判断基于 `references/blood_test_reference.json`
   - 不使用其他来源或个人知识

3. **患者安全优先 Prioritize Patient Safety**
   - 存疑时建议咨询医生
   - 强调本技能提供的是辅助工具而非医疗诊断

### 台账管理规范 / Ledger Management

1. **数据格式统一**
   - 统一单位标准
   - 数值保留合适精度
   - 日期格式统一为YYYY-MM-DD

2. **完整可追溯**
   - 每条记录包含检查日期和医院
   - 保留原始检测值和参考范围
   - 记录录入时间戳

3. **便于展示导出**
   - 表格排版整齐，适合直接截图
   - 关键数据突出显示
   - 支持医生复诊时快速查看

### 沟通风格 / Communication Style

1. **清晰且富有同理心 Clear and Compassionate**
2. **以行动为导向 Action-Oriented**
3. **全面且有层次 Comprehensive but Organized**

## 🔑 本地配置说明（重要）

### 飞书集成配置

本 skill 支持将报告数据自动录入飞书电子表格。使用此功能需要配置飞书应用凭证：

#### 步骤1：获取飞书应用凭证

1. 登录 [飞书开放平台](https://open.feishu.cn/)
2. 创建或进入你的自建应用
3. 在「凭证与基础信息」页面获取：
   - **App ID** (格式: `cli_xxxxxxxxx`)
   - **App Secret** (格式: `xxxxxxxx`)

#### 步骤2：创建本地配置文件

在 **skill 同级目录** 创建 `config.json` 文件，内容如下：

```json
{
  "app_id": "cli_你的AppID",
  "app_secret": "你的AppSecret",
  "spreadsheet_token": "你的电子表格Token",
  "sheets": {
    "blood_routine": {
      "name": "血常规",
      "enabled": true,
      "columns": ["日期", "医院", "WBC", "ANC", "PLT", "Hb", "异常", "建议"]
    },
    "ct_report": {
      "name": "CT报告",
      "enabled": true,
      "columns": ["日期", "医院", "检查类型", "原发灶", "转移灶", "总体评价", "医生建议"]
    }
  },
  "settings": {
    "deduplication": { "enabled": true, "key_fields": ["date", "hospital", "report_type"] },
    "emergency_warning": true
  }
}
```

#### 步骤3：获取电子表格 Token

1. 打开或创建一个 [飞书电子表格](https://p16aafnobs.feishu.cn/sheets)
2. 从浏览器地址栏复制 Token: `https://p16aafnobs.feishu.cn/sheets/{TOKEN}`
3. 填入 `spreadsheet_token` 字段

### 配置文件位置

| 环境 | 配置路径 |
|------|----------|
| CodeBuddy Skill | `~/.codebuddy/skills/病历管家/config.json` |
| 或 | skill 目录下的 `config.json` |

> **安全提示**: `config.json` 包含敏感凭证，已加入 `.gitignore`，**不会被推送到 GitHub**

### 不配置飞书？

如果不需要自动归档到飞书表格，skill 仍可正常使用所有识别和分析功能，只是数据会以文本形式输出而非写入表格。
