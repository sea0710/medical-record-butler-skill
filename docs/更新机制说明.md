# Skill更新机制说明

## 🔄 更新架构

### 方案概述

我们采用 **GitHub Releases + 自动检测** 的更新机制:

```
管理员(您)
    ↓ 更新Skill代码
    ↓ 打包新版本
    ↓ 发布到GitHub Releases
    ↓
GitHub仓库
    ↓ 用户检测更新
    ↓
用户端
    ↓ 下载新版本
    ↓ 重新安装
    ↓ 使用最新版本
```

---

## 📦 版本管理

### 版本号规范

采用语义化版本号: `v主版本.次版本.修订号`

```
v2.0.0
│ │ │
│ │ └─ 修订号: Bug修复,小改进
│ └─── 次版本: 新功能,向后兼容
└───── 主版本: 重大更新,可能不兼容
```

### 发布流程

每次您更新Skill时:

**步骤1: 更新代码**
```bash
# 修改Skill文件
# 测试新功能
# 确保稳定可用
```

**步骤2: 更新版本号**
```yaml
# 在SKILL.md中更新
version: v2.1.0
last_update: 2026-03-30
```

**步骤3: 打包**
```bash
# 使用skill-creator打包
python3 scripts/package_skill.py [skill路径] [输出目录]
```

**步骤4: 发布到GitHub**
```bash
# 创建Git标签
git tag v2.1.0
git push origin v2.1.0

# 上传到Releases
# 添加更新日志
```

## 🔍 用户端更新

### 自动检测机制

用户可以:

**方式1: 手动检查**
```
用户询问: "检查chemo-blood-monitor skill是否有更新?"
AI: 正在检查...
    发现新版本 v2.1.0
    更新内容:
    - 添加趋势图表功能
    - 修复飞书API问题
    
    是否下载安装?
```

**方式2: 定期提醒**
```
AI提示: 您的chemo-blood-monitor skill有新版本可用
       当前版本: v2.0.0
       最新版本: v2.1.0
       建议更新以获得最新功能
```

### 更新步骤

用户更新Skill:

```
1. 下载最新版本
   从GitHub Releases下载 chemoblood-monitor-v2.1.0.zip

2. 卸载旧版本
   在CodeBuddy中卸载旧版skill

3. 安装新版本
   安装新下载的zip文件

4. 验证更新
   AI会显示当前版本信息
```

## 🔐 配置文件管理

### 配置文件位置

```
chemo-blood-monitor/
├── config/
│   ├── feishu_config.json      # 用户飞书配置
│   └── user_settings.json      # 用户个人设置
```

### 配置保留策略

**升级时保留**:
- ✅ 飞书App ID和App Secret
- ✅ 文档ID
- ✅ 个人设置

**升级时重置**:
- ❌ Skill核心代码
- ❌ 参考文档

**解决方案**:
配置文件独立存储,不包含在skill包中,升级时自动保留。
